WeHelp Academy 第七屆申請

Submit by 金彥良 JIN YEN-LIANG

01 個人簡介和申請動機 +

前一份及目前這一份工作，產業皆是網路媒體，雖然主要工作內容是視覺設計，但也接觸到網頁設計，進而對於前端技術感興趣。

雖然對於前端技術最感興趣的部分仍是切版、動態特效技術（GSAP、Motion、Three.js等Library），但也在幾次實作中踩坑，碰到效能上的瓶頸，以及更為複雜的使用者互動（如需儲存使用者的資料），覺得自己若想要更進一步，必須要更了解網頁前後端、伺服器部署等基礎。

因此決定今年九月底離職，全職投入學習與轉職。

02 曾經做過哪些軟體工程技術相關的學習？ +

目前有一個個人網站，上面的網站作品有部分是HTML+CSS+JS單頁式的切版，有部分是透過Webflow這樣的架站平台去完成。

去年曾報名過六角學院的軟體工程師體驗營，算是第一次比較有系統地學習HTML+CSS+JS的基礎，同時間也進行了 JavaScript 30 的練習，但大多是跟著Youtube上的解說自己實作一遍。

隨後在去年9-11月，上了台大資訊系統訓練班的「AI應用網站開發入門 HTML CSS React」，是第一次接觸到前端框架，從課程中初步了解React Component和Hook(useState, useEffect)的概念。

03 如果參與這個訓練，會怎麼安排學習時間？ +

目前計劃是全職投入訓練，但在第一階段會跟一個自己接案的專案有所重疊（該專案會到11月中結束）。目前考量第一階段的前端三劍客應該算有一些基礎，所以會把時間優先用於Python、FastAPI、MySQL等尚不熟悉的部分。

整體而言，學習時間主要會是週一到週五白天時間，計劃是去家裡附近星巴克或圖書館進行學習（在家較容易分心）；晚上及假日需陪伴家人，但每天應該仍能抽出1-2小時（假日視情況2-4小時）做碎片式的學習，可能會利用較碎片的時間，透過YT, CodeCademy 等管道吸收與練習小塊的知識。

04 軟體技術日新月異，如何確定投入的領域是正確有回報的？ +

我的個性偏向先做再說、邊走邊修正。回顧大學以來，我的人生方向已經做了非常多次的修正，從簡報慢慢到設計，再從設計開始接觸到前端工程，那些我已經不再投入的領域，並不是什麼走錯的路，而是當下我要前進的、我要克服的事項。那些經驗，我覺得也或多或少幫助我去克服下一個的課題。

所以我會覺得，以未來一年的時間為尺度，只要我能從只能做簡單的HTML,CSS,JS網站，進步到能獨立完成一個電商網站、個人專題，並且具備自己查找問題根源、解決、優化的能利，即便未來我沒有繼續在這條路上前進，我也不覺得這樣的訓練是沒有回報的。

05 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？ +

因為一些緣故，目前算是獨力在建置公司的一個網站，雖然只是單頁式網站的架構，但因為一開始預期會有工程師協助，所以設想了一些比較進階、有效能負擔的動畫效果。而後來因為公司工程師的人力缺額無法被即時補足，所以變成我需要自行完成後續；雖然有找了一個外部顧問可以諮詢，但因為他無法直接幫忙寫code（時間關係只能做debug），所以在前兩月的時候感到非常大的壓力，特別是React很多Hooks都還不熟悉、幾乎都是我描述需求請AI寫出來，但同事在檢視的時候，又會提出一些不一樣的想法，再加上AI偏向是完成我的需求而無法很好地顧及整個專案架構，所以前端效能很差。

這個過程或多或少推進了我離職的決定，但除了想離職以外，也好幾度在半夜睡不著、即便睡著都是夢到在解bug的情境，甚至覺得還是我徹底擺爛、一走了之就好。

雖然現在還沒完全結束這個專案，手機上也還有一些效能問題，但至少已經算是看得到盡頭了。坦白說，度過這些情緒的辦法，就是深呼吸、把注意力拉回自己可以控制的要素上、思考更實務的解決辦法——譬如將視覺特效大幅減少、手機版用現成套件完成等等。這個時間點回顧，我自己的檢討是，應該更密切地跟外部顧問以及內部工程師討論，即便他們沒辦法直接幫我解決問題，也都可以點出一些我不知道我不知道的事情，譬如前兩天跟內部工程師提到相關問題，他們就說有一個方法是profiling、可以去查查相關的方法。

06 關於這份申請網頁，分享一個開發時的技術心得 +

前陣子剛好看一個工程師在做直播，當時好像是他在教他的學生，怎麼從零到一自己做出一個Tab和Accordion的元件。這次就想說也把申請的問答做成Accordion的方式，只不過我想嘗試不用JS，而是用<details><summary>這組HTML原生的tag和CSS來實現，覺得效果意外的不錯。

過去對於HTML的元素只有掌握最基礎的那幾個，但其實打開W3School就可以看到非常多元素，像是過去圖片和圖說，我就是用<div>包住<img>和<p>，不過因為這次查詢，才發現其實有原生的<figure>與<figcaption>可以使用；雖然畫面上的實現沒有差異，但對於SEO成效可能會更好。

07 如何看待人工智慧的發展對軟體工程師的影響？ +

我覺得AI帶來的影響是讓「基礎的程式邏輯」變成一個技能，而不再是特定角色才具備的能力。這有點類似單眼攝影演進到數位相機、手機攝影，一個能力的進入門檻大幅降低，而致使許多原本的需求方也開始具備相關的基礎能力、自行完成攝影工作；也因為攝影門檻的降低，致使原本沒有需求（或因成本而止步）的人們也開始接觸攝影。

但這並未消滅攝影師的存在，反而讓攝影進一步蓬勃發展，專業攝影師開始運用更複雜、進階的技術，並且與美術、造型、燈光、設計等領域整合，致使要名符其實成為一位「攝影師」的門檻變得更高。

因此我自己是持樂觀態度，雖然這在一定時間內一定會影響市場上的職缺，但恐怕也會帶動其他需求的成長，行銷、設計、財會等職務會開始出現需要同時具備原專業與程式能力的職缺與人才，而要專職作為軟體工程師的門檻變得更高。
